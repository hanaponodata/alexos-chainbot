# ChainBot GUI — Next-Gen Frontend

## Vision
A glassmorphic, animated, agentic interface that feels magical, fast, and more delightful than ChatGPT or Cursor.

## Tech Stack
- React + Vite + TypeScript
- Zustand (state)
- Chakra UI (customized)
- Framer Motion (animation)
- Monaco Editor
- MSW (Mock Service Worker) for all backend data

## Mock-Driven Development
- All API calls are intercepted and mocked with MSW.
- Fake agents, chats, code suggestions, logs, and workflows.
- Simulate delays, errors, and agent typing for realism.
- Document every endpoint and payload as you go—this becomes your future API contract.

## Game Plan
1. Build a beautiful, glassy, animated UI with all features mocked.
2. Demo and iterate on UX, microinteractions, and polish.
3. Document all mock endpoints/types for backend handoff.
4. Integrate real backend only after the frontend is spectacular.

## How to Run
1. `npm install`
2. `npm run dev`

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type-aware lint rules:

```js
export default tseslint.config([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...

      // Remove tseslint.configs.recommended and replace with this
      ...tseslint.configs.recommendedTypeChecked,
      // Alternatively, use this for stricter rules
      ...tseslint.configs.strictTypeChecked,
      // Optionally, add this for stylistic rules
      ...tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```

You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from 'eslint-plugin-react-x'
import reactDom from 'eslint-plugin-react-dom'

export default tseslint.config([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...
      // Enable lint rules for React
      reactX.configs['recommended-typescript'],
      // Enable lint rules for React DOM
      reactDom.configs.recommended,
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```
